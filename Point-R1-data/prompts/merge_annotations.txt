## 任务
请基于图像和标注内容，为每个子部件生成最终的综合标注。

### 整合原则

1. **最优视角选择**: 对于每个部件，选择能**最清楚、最完整**地观察到该部件的视角作为主要参考
   - 部件完全可见、无遮挡的视角优先
   - 部件占据较大区域、细节清晰的视角优先
   - 避免使用部件被遮挡或仅露出一小部分的视角

2. **视角互补**: 主视角不足的信息可从其他视角补充
   - 如果某视角能看到主视角看不到的细节，可以补充进来
   - 综合多个视角的观察，形成完整描述

3. **信息筛选**: 优先采信最优视角的描述
   - 如果不同视角的描述有冲突，以最优视角为准
   - 不要简单拼凑各视角的描述，而是有主次地整合

### 输出格式
```json
{{
    "merged_annotations": [
        {{
            "som_id": 1,
            "color": "SoM颜色名称",
            "name": "最终确定的名称",
            "caption": "综合多视角信息的详细描述（30-80字）",
            "best_view": "最优视角（如：视角1、视角2等）",
            "reasoning": "简要描述选择该视角的原因和整合策略"
        }},
        ...
    ],
    "annotation_notes": "整体标注过程中的注意事项或不确定性说明"
}}
```

### 重要：描述内容要求
- **专注于 3D 对象本身**：最终 caption 应描述部件的形状、结构、功能、材质等固有属性
- **忽略视角相关信息**：不要在 caption 中提及任何视角、角度、可见性等信息
- **描述应具有普遍性**：生成的描述应该是对该部件的客观、全面的描述，而非特定视角下的观察结果
